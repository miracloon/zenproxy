# v0.3.2 Design — 客户端端口配置增强

> **状态**：design 草稿，待用户确认范围后进入 plan。

---

## 候选范围（来自 INTENT.md Fork 增量意图）

以下为 INTENT.md 中尚未实现的功能项，按优先级和耦合度排列：

### §2 客户端端口配置增强

- 允许自定义代理池端口范围（源项目硬编码 20001-30000，fork 默认改为 60001-65535）
- 允许客户端端口与服务端端口同步配置
- **涉及修改**：`sing-box-zenproxy/experimental/clashapi/proxy_manage.go`（Go）
- **复杂度**：中 — 需修改 PortPool 初始化逻辑，从配置读取范围

### §3 服务端认证方式扩展

- 在现有 Linux.do OAuth 基础上，新增用户名/密码认证
- **涉及修改**：`src/api/`（Rust）、`src/config.rs`
- **复杂度**：中

### §4 配置方案优化

- 统一配置文件方案
- **依赖**：取决于 §2、§3 的实现方式
- **建议**：与 §2、§3 一起做

### §5 客户端 Dashboard

- 单页 HTML Dashboard，嵌入 Clash API
- **涉及修改**：`sing-box-zenproxy/experimental/clashapi/`（Go）
- **复杂度**：中到高
- **建议**：独立版本
